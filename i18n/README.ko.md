## 존재 이유

레스토랑, 펍, 와인 숍, 슈퍼마켓: 문제는 항상
동일합니다. **수십 가지 선택지가 있지만, 어떤 것이 진정으로
내 취향에 맞을지 확신할 수 없습니다.**

웨이터에게 물어보거나, 휴대폰에서 다섯 개의 탭을 열어보거나,
가장 예쁜 이름이 가장 좋은 것이길 바랄 수도 있습니다. 또는
AI 에이전트에게 물어볼 수도 있습니다 — 그러면 에이전트는 이미
당신의 취향을 알고 있습니다.

**drinks-sommelier**는 모든 AI 에이전트가 개인
소믈리에 역할을 하도록 가르칩니다. 에이전트는:

1. 한 번만 당신의 취향 선호도를 학습합니다
2. 모든 입력을 분석합니다: 선반 사진, 디지털 메뉴, 사용자가
   작성한 이름 목록
3. 각 제품에 대한 최신 정보를 검색합니다
4. 각 제품을 당신의 취향 프로필과 비교합니다
5. 정확히 무엇을 선택해야 하는지, 그리고 그 이유를 알려줍니다

모든 과정이 투명하고, 정직하며, 무료이고, 아무것도 지어내지 않습니다.

## 장점

- **시간이 지남에 따라 취향을 학습합니다.** 스킬을 사용할수록 에이전트는
  당신의 미각을 더 잘 알게 됩니다. 각 상호작용이 프로필을 정교하게 만듭니다.

- **맥주와 와인이 하나의 스킬에.** 두 개의 개별 스킬을
  설치할 필요가 없습니다. 에이전트는 두 카테고리를 모두 처리하며,
  분석 대상을 자동으로 인식합니다.

- **이미지와 텍스트 모두 분석.** 선반, 메뉴, 와인 리스트의
  사진을 찍거나 목록을 작성하세요. 에이전트는 관련 제품
  (맥주와 와인)만 추출하고 나머지는 모두 무시합니다.

- **환각 현상 없음.** 에이전트는 사전 학습 지식을
  사용하는 것이 **절대적으로 금지**됩니다. 각 제품은
  평가 전에 항상 웹에서 즉시 조사됩니다.

- **선호도 지수 0-100%.** 각 제안에는 제품이 당신의 취향에
  얼마나 잘 맞는지 나타내는 명확한 점수가 있습니다.
  애매모호함은 없습니다.

- **영구적인 취향 프로필.** 당신의 선호도는
  읽고 편집할 수 있는 텍스트 파일에 저장됩니다.
  세션 간에 손실되지 않습니다.

- **에이전트 주도의 설정.** 처음에는 에이전트가
  적절한 질문을 통해 당신의 취향을 파악하므로,
  수동 설정에 직면할 필요가 없습니다.

- **오픈소스, MIT 라이선스.** 무료, 구독 없음,
  API 키 없음, 제한 없음. 사용, 수정, 배포가 가능합니다.

- **모든 AI 에이전트에서 작동.** OpenCode용으로 작성되었지만
  OpenClaw, Hermes Agent, Claude Code, Cursor,
  GitHub Copilot 및 기타 모든 에이전트로 쉽게 변환 가능합니다.
  이 README를 에이전트에 보여주면 간단하고 자동으로 변환됩니다.

- **완전히 사용자 정의 가능.** SKILL.md는 텍스트 파일입니다.
  규칙을 추가하고, 기준을 수정하고, 워크플로에 맞게 조정하세요.

## 작동 방식

프로세스는 5단계로 나뉘며, 쿼리할 때마다 에이전트가
순서대로 실행합니다.

### 1단계 — 취향 프로필 설정 (최초 1회)

스킬이 처음 로드될 때, 에이전트는 SKILL.md 내의
"사용자의 맥주 취향" 및 "사용자의 와인
취향" 문단이 이미 작성되었는지 확인합니다.

작성되지 않은 경우, 에이전트는 `data/SETUP.md` 파일을 엽니다. 여기에는 다음이 포함됩니다:
- 선호도를 수집하기 위한 안내 질문 (단맛/쓴맛, 알코올 도수,
  좋아하는 스타일과 싫어하는 스타일 등)
- 올바르게 작성된 문단의 예시
- 데이터가 입력된 `data/` 파일의 예시

에이전트가 질문을 하고, 답변을 수집하며, SKILL.md의
두 취향 문단을 수정합니다. 설정 중에 사용자가
특정 제품을 언급하면, 에이전트는 이를 해당
`data/` 파일에 기록합니다 (선택 사항이지만
프로필을 정교하게 만드는 데 유용합니다).

원한다면 문단을 직접 작성할 수도 있습니다.

`data/known-preferred-beers.md` 및 `data/known-preferred-wines.md`
파일은 선택 사항이며, 과거에 이미 평가한 특정 제품이 포함됩니다.
이 파일이 완전할수록 에이전트가 당신의 미묘한 차이를 더 잘 이해합니다.

### 2단계 — 입력 분석

에이전트는 당신이 무엇을 어떤 형식으로 제공했는지 식별합니다:

- **텍스트**: 작성된 목록에서 맥주와 와인의 이름을 추출합니다
- **이미지**: 사진(선반, 메뉴, 와인 리스트)을 분석하고
  맥주 또는 와인인 제품만 인식합니다.
  완전히 무시하는 항목: 증류주, 리큐르, 스낵, 식사,
  디저트, 탄산음료, 액세서리. 맥주나 와인이 아닌 모든 것.
- **혼합**: 텍스트와 이미지를 모두 보낸 경우, 모든 것을 분석합니다.

입력이 명확하지 않은 경우, 에이전트는 진행하기 전에 질문합니다.

### 3단계 — 웹 조사 (환각 방지)

에이전트는 제품을 평가하기 위해 사전 학습 지식을
사용하는 것이 **절대적으로 금지**됩니다. 해당 지식은
구식이거나 부정확하거나, 더 나쁘게는 지어낸 것일 수 있습니다.
따라서 식별된 모든 제품에 대해 대상 웹 검색을
수행합니다.

에이전트가 검색하는 내용:

| 카테고리 | 수집 정보 |
|---|---|
| **맥주** | 스타일, 알코올 도수, 단맛, 쓴맛(IBU), 바디, 아로마 노트, 원재료 |
| **와인** | 포도 품종, 원산지 명칭, 생산 지역, 빈티지, 바디, 산도, 탄닌, 알코올 도수, 단맛 특성 |

제품을 온라인에서 찾을 수 없는 경우, 에이전트는 정직하게
그 사실을 알리고 특성을 지어내지 않습니다.

### 4단계 — 평가

에이전트는 각 제품을 당신의 취향 프로필과 비교합니다. 프로필에는
두 가지 수준이 있습니다:

1. **취향 규칙** — 취향 문단에 작성된 일반 지침
   (예: "단 맥주", "너무 달지 않은
   와인", "알코올 도수 ≤ 9°"). 이는 구속력이 있습니다.
2. **data/ 파일의 제품** — 당신이 좋아했거나 싫어했던
   맥주와 와인의 구체적인 예시. 이는 취향 규칙을
   보정하는 데 사용됩니다 (예: 맥주 X를 좋아하고 Y를 싫어한다면,
   에이전트는 당신이 말하는 "단맛"과 "쓴맛"의 의미를 이해합니다).

에이전트는 **0에서 100%까지의 선호도 지수**를 할당합니다:

| 지수 | 의미 |
|---|---|
| 90-100% | 당신의 취향에 완벽한 제품 |
| 70-89% | 우수, 약간의 차이 있음 |
| 50-69% | 수용 가능하지만 최적은 아님 |
| 30-49% | 당신의 취향에 맞지 않음 |
| 0-29% | 피해야 함 |

식사 맥락이 지정된 경우, 에이전트는
푸드 페어링도 평가합니다 (부차적 우선순위).

### 5단계 — 추천

에이전트는 답변을 명확하게 구성합니다:

1. **첫 번째 선택**: 선호도 지수와 함께 최적의 제품,
   그리고 그것이 당신의 취향에 적합한 이유 설명
2. **대안** (있는 경우): 선호도가 높은 순서대로
3. **페어링**: 제품에 대한 음식 페어링 제안
4. **피해야 할 제품**: 당신의 취향에 명확히 맞지 않는
   제품과 그 이유 설명

## 설정

스킬이 작동하려면 당신의 취향을 알아야 합니다.
두 가지 방법으로 설정할 수 있습니다:

### 안내 설정 (권장)

스킬을 설치한 후, 에이전트에게 "drinks-sommelier
스킬을 설정해줘"와 같이 요청하세요. 에이전트는 SKILL.md를
로드하고, 아직 작성되지 않은 취향 문단을 찾아
`data/SETUP.md`를 사용하여 초기 설정을 안내합니다.

또는 단순히 맥주나 와인 추천을 요청하세요:
에이전트가 스킬이 초기화되지 않았음을 자동으로 감지하고
적절한 질문을 할 것입니다.

### 수동 설정

`SKILL.md`를 열고 다음 문단을 작성하세요:

**사용자의 맥주 취향**
```
달고 쓰지 않은 맥주를 선호합니다.
알코올 도수는 9%를 초과하지 않아야 합니다.
벨기에 스타일과 과일 맥주를 좋아합니다.
IPA, 스타우트, 너무 쓴 맥주는 싫어합니다.
```

**사용자의 와인 취향**
```
너무 달지 않은 레드 와인을 선호합니다.
화이트 와인은 베르멘티노와 소비뇽을 좋아합니다.
산도와 신선함이 좋은 와인을 좋아합니다.
파시토 또는 강화 와인은 싫어합니다.
```

또한 `data/known-preferred-beers.md` 및
`data/known-preferred-wines.md` 파일에 이미 시음하고
평가한 특정 제품을 기록할 수 있습니다. 이 부분은 선택 사항이지만
프로필을 정교하게 만드는 데 유용합니다.

**예시 — `data/known-preferred-beers.md`**
```
# 좋아하는 맥주
- Kwak
- Triple Karmeliet
- La Trappe Blond
- Flea Margherita

# 싫어하는 맥주
- 모든 IPA
- 스타우트
- 알코올 도수 9% 초과 맥주
```

**예시 — `data/known-preferred-wines.md`**
```
# 좋아하는 와인
- Vermentino Costamolino
- Syrah Soraia Casale Valle Chiesa
- Rebeca Firriato

# 싫어하는 와인
- 파시토 와인
- 강화 와인
```

### 시간에 따른 업데이트

제품에 대한 평가를 내릴 때마다 ("이거 좋아요",
"이거 싫어요"), 에이전트가 `data/` 파일을 자동으로 업데이트합니다.
반면 SKILL.md 문단의 취향 규칙은 향후 평가에
구속력을 가지므로 명시적인 확인이 있을 때만
업데이트됩니다.

## 사용 예시

### 시나리오 1 — 와인 선반 (이미지)
와인 샵 선반의 사진을 찍습니다. 에이전트가 와인을 식별하고,
모두 웹에서 조사한 후 프로필과 비교합니다. 어떤 것을
선택해야 하는지, 그 이유, 그리고 무엇과 곁들여야 하는지 알려줍니다.

### 시나리오 2 — 맥주 메뉴 (이미지)
펍의 맥주 메뉴 사진을 찍습니다. 에이전트는 칵테일,
증류주 등을 무시하고 맥주만 분석하여 각 맥주의 정보를
검색합니다. 당신의 취향에 가장 적합한 맥주를 제안합니다.

### 시나리오 3 — 혼합 텍스트 목록
"이런 와인들이 있어요: Chianti Classico, Vermentino Costamolino,
Amarone. 그리고 이런 맥주들: Kwak, 지역 수제 IPA, Leffe Blonde."
라고 작성합니다. 에이전트는 두 카테고리를 모두 평가하고
전체적으로 최상의 선택이 무엇인지 알려줍니다.

### 시나리오 4 — 목록 없는 요청
"저녁 식사용 와인을 사야 하는데, 뭐가 좋을까요?"
에이전트는 지어내지 않습니다. 무엇을 구할 수 있는지,
어떤 종류의 저녁인지, 이미 염두에 둔 와인이 있는지 묻습니다.

### 시나리오 5 — 새로운 선호도 표현
추천 후 "그 맥주 정말 마음에 들었어요"라고 말합니다.
에이전트는 그것을 즐겨찾기에 추가하고 다음 번에
더 정확해지도록 프로필을 업데이트합니다.

---

## 이 스킬이 하지 않는 일

- **전문적인 시음 평가를 제공하지 않습니다.** 제안은
  객관적 데이터(스타일, 알코올 도수,
  쓴맛)와 명시된 선호도에 기반합니다. 공식적인 상황에서
  인간 소믈리에를 대체하지 않습니다.

- **다른 음료는 처리하지 않습니다.** 증류주, 칵테일, 리큐르,
  탄산음료, 무알코올 음료 — 맥주나 와인이 아닌 것은
  모두 무시됩니다.

## 제안 (선택 사항)

**drinks-sommelier**는 에이전트가 기본적으로 지원하는
모든 웹 검색 도구에서 작동합니다. 에이전트에 이미
신뢰할 수 있는 웹 검색 기능이 있다면 추가로 필요한 것은 없습니다.

하지만 매번 철저하고 안티봇 저항성이 있는 검색을
보장하려면 (특히 제품 페이지가 안티봇 시스템 뒤에 있을 때 유용),
함께 제공되는 **[browser-search](https://github.com/Johell1NS/browser-search)**
스킬 설치를 고려하세요. 수십 개의 엔진을 동시에 검색하여
표준 도구가 어려움을 겪을 때 에이전트가 제품 정보를
찾을 수 있도록 도와줍니다.
