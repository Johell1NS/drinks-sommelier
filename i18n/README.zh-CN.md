> 一个专精于啤酒和葡萄酒的虚拟侍酒师，它能了解你的个人
> 口味，并从任何货架、菜单或列表中推荐最佳选择。**免费、开源、自托管。**

---

## 存在的原因

在餐厅、酒吧、酒类商店、超市：问题总是
相同的。**几十种选择，却无法确定**哪一款真正
适合你的口味。

你可以询问服务员，在手机上打开五个标签页，或者
指望最好听的名字也是最佳选择。或者你可以问问
你的AI代理——而它已经了解你的喜好。

**drinks-sommelier** 教会任何AI代理扮演个人
侍酒师的角色。该代理会：

1. 一劳永逸地了解你的口味偏好
2. 分析任何输入：货架照片、电子菜单、用户
   书写的名称列表
3. 搜索每种产品的最新信息
4. 将每种产品与你的口味档案进行比较
5. 准确告诉你该选什么——以及为什么

所有过程透明、诚实、免费，且绝不凭空捏造。

## 优势

- **随时间学习你的口味。** 你使用该技能越多，代理就越了解
  你的味觉。每次互动都会完善口味档案。

- **啤酒和葡萄酒合二为一。** 你无需安装两个独立的
  技能。代理能处理这两个类别，并自动识别
  正在分析的对象。

- **支持图像和文本分析。** 拍摄货架、菜单、
  酒单的照片。或者写一个列表。代理只会提取
  相关的产品（啤酒和葡萄酒），忽略其他一切。

- **绝不产生幻觉。** 代理被绝对禁止使用
  其自身的预训练知识。每种产品在评估前
  都会立即通过网络进行查询。

- **偏好指数 0-100%。** 每个推荐都有清晰的
  评分，说明该产品与你的口味匹配程度。没有
  模棱两可。

- **持久的口味档案。** 你的偏好保存在
  可读可编辑的文本文件中。不会在
  不同会话之间丢失。

- **代理引导的设置。** 首次使用时，代理会问你
  合适的问题来了解你的口味，无需你面对
  手动配置。

- **开源，MIT许可证。** 免费，无订阅，无
  API密钥，无限制。你可以使用、修改、分发。

- **适用于任何AI代理。** 为OpenCode编写，但
  可轻松转换为OpenClaw、Hermes Agent、Claude Code、Cursor、
  GitHub Copilot及任何其他代理。将此README展示给你的
  代理即可实现简单自动的转换。

- **完全可定制。** SKILL.md是一个文本文件。
  添加规则、修改标准、适应你的工作流程。

## 工作原理

该过程分为5个阶段，每次查询时由代理按顺序执行。

### 阶段一 — 口味档案设置（一次性）

首次加载该技能时，代理会检查 SKILL.md 中的
"用户啤酒口味"和"用户葡萄酒口味"段落是否
已经填写。

如果尚未填写，代理会打开 `data/SETUP.md` 文件，其中包含：
- 指导性问题以收集偏好（甜/苦、酒精度、
  喜欢和不喜欢的风格等）
- 正确填写段落的示例
- 已填充的 `data/` 文件示例

代理会问你问题、收集答案并修改 SKILL.md 中的
两个口味段落。如果在设置过程中用户提到特定产品，代理
会将其记录在相应的 `data/` 文件中（可选
操作，但有助于完善档案）。

如果你愿意，也可以手动填写这些段落。

`data/known-preferred-beers.md` 和 `data/known-preferred-wines.md`
文件是可选的，包含你过去已经评估过的具体产品。
它们越完整，代理就越能理解你的细微差别。

### 阶段二 — 输入分析

代理识别你提供的内容及其格式：

- **文本**：从书面列表中提取啤酒和葡萄酒的名称
- **图像**：分析照片（货架、菜单、酒单）并
  仅识别属于啤酒或葡萄酒的产品。
  完全忽略：烈酒、利口酒、小吃、餐食、
  甜点、汽水、配件。任何非啤酒或葡萄酒的东西。
- **混合**：如果你同时发送文本和图像，它会分析所有内容。

如果输入不明确，代理会在继续前先询问。

### 阶段三 — 网络查询（防幻觉）

代理**被绝对禁止**使用其自身的预训练知识
来评估产品。这些知识可能已过时、
不准确，或更糟：凭空捏造。因此，它会对每个
已识别的产品进行有针对性的网络搜索。

代理搜索的内容：

| 类别 | 收集的信息 |
|---|---|
| **啤酒** | 风格、酒精度、甜度、苦度(IBU)、酒体、香气特征、成分 |
| **葡萄酒** | 葡萄品种、产区、生产区域、年份、酒体、酸度、单宁度、酒精度、甜度特征 |

如果某个产品在网上找不到，代理会如实说明
且不会编造特征。

### 阶段四 — 评估

代理将每种产品与你的口味档案进行比较，该档案包含
两个不同层级：

1. **口味规则** — 你在口味段落中写下的
   通用指南（例如"甜啤酒"、"不太甜的
   葡萄酒"、"酒精度 ≤ 9°"）。这些规则具有约束力。
2. **data/ 文件中的产品** — 你喜欢或不喜欢的
   啤酒和葡萄酒的具体示例。它们用于校准
   口味规则（例如，如果你喜欢啤酒X而讨厌Y，代理
   就能理解你所说的"甜"和"苦"的含义）。

代理会给出一个**0 到 100% 的偏好指数**：

| 指数 | 含义 |
|---|---|
| 90-100% | 非常适合你口味的产品 |
| 70-89% | 优秀，略有差异 |
| 50-69% | 可接受，但非最佳 |
| 30-49% | 不太适合你的口味 |
| 0-29% | 应避免 |

如果你指定了用餐场景，代理还会评估
食物搭配（次要优先级）。

### 阶段五 — 推荐

代理会清晰结构化地给出回答：

1. **首选**：最佳产品，附有偏好指数
   和解释为何适合你的口味
2. **备选方案**（如有）：按偏好降序排列
3. **搭配建议**：产品与食物搭配的建议
4. **应避免的产品**：明显不符合
   你口味的产品，附有解释

## 安装

你可以通过两种方式安装该技能：

### 使用 npx skills（推荐）

```bash
npx skills add Johell1NS/drinks-sommelier --skill drinks-sommelier
```

### 使用 git clone

```bash
git clone https://github.com/Johell1NS/drinks-sommelier.git
```

然后将 `skills/drinks-sommelier/` 文件夹复制（或符号链接）到
你代理的技能目录中。

> **注意：** 与 `npx skills add` 不同，git clone 不知道你的
> 代理技能存储在何处。你需要将技能放置在代理的
> 正确路径中（例如，OpenCode 使用 `~/.config/opencode/skills/`，
> 其他代理可能不同）。放置好后，代理会自动
> 检测到它。

## 配置

该技能需要了解你的口味才能工作。你可以通过
两种方式进行配置：

### 引导式配置（推荐）

安装技能后，向你的代理提问，例如
"帮我配置 drinks-sommelier 技能"。代理会加载
SKILL.md，找到尚未填写口味段落的位置，并使用
`data/SETUP.md` 引导你完成初始设置。

或者，直接请求啤酒或葡萄酒推荐：
代理会自动检测到该技能尚未初始化，
并向你提出适当的问题。

### 手动配置

打开 `SKILL.md` 并填写以下段落：

**用户啤酒口味**
```
我喜欢甜而不苦的啤酒。
酒精度不得超过 9%。
我喜爱比利时风格和水果啤酒。
我不喜欢IPA、世涛和过于苦涩的啤酒。
```

**用户葡萄酒口味**
```
我偏爱不太甜的红葡萄酒。
白葡萄酒方面，我喜欢维蒙蒂诺和长相思。
我喜爱酸度适中、口感清新的葡萄酒。
我不喜欢帕赛托或加强型葡萄酒。
```

你也可以填充 `data/known-preferred-beers.md` 和
`data/known-preferred-wines.md` 文件，记录你已经
品尝和评估过的具体产品。这部分是可选的，但有助于
完善口味档案。

**示例 — `data/known-preferred-beers.md`**
```
# 喜欢的啤酒
- Kwak
- Triple Karmeliet
- La Trappe Blond
- Flea Margherita

# 不喜欢的啤酒
- 任何IPA
- 世涛
- 酒精度超过 9% 的啤酒
```

**示例 — `data/known-preferred-wines.md`**
```
# 喜欢的葡萄酒
- Vermentino Costamolino
- Syrah Soraia Casale Valle Chiesa
- Rebeca Firriato

# 不喜欢的葡萄酒
- 帕赛托葡萄酒
- 加强型葡萄酒
```

### 随时间更新

每当你对某个产品做出评价（"我喜欢这个"、
"我不喜欢这个"），代理会自动更新
`data/` 文件。而 SKILL.md 段落中的口味规则则
只有在你明确确认后才会更新，因为它们对
未来的评估具有约束力。

## 使用示例

### 场景一 — 葡萄酒货架（图片）
你拍摄了一张酒类商店货架的照片。代理会识别葡萄酒，
全部在网上查询，与你的口味档案进行对比。它告诉你
该选哪一款、为什么选它，以及搭配什么食物。

### 场景二 — 啤酒菜单（图片）
你拍摄了一张酒吧啤酒菜单的照片。代理会忽略鸡尾酒、
烈酒等……仅分析啤酒，搜索每种啤酒的信息。
它推荐最适合你口味的啤酒。

### 场景三 — 混合文本列表
你写下："我有这些葡萄酒：Chianti Classico、Vermentino Costamolino、
Amarone。还有这些啤酒：Kwak、本地精酿IPA、Leffe Blonde。"
代理会评估两个类别，并告诉你整体上最佳的
选择是什么。

### 场景四 — 无列表的请求
"我需要买一瓶酒用于晚餐，你有什么推荐？"代理不会
凭空编造。它会问你有什么可选、是什么类型的晚餐，
以及你是否已经有了一些想法。

### 场景五 — 表达新的偏好
在你得到推荐后说："我真的很喜欢那款啤酒。"
代理会将其添加到你的收藏夹中，并更新口味档案，
以便下次更加精准。

## 该技能不做什么

- **不提供专业的品酒评价。** 推荐
  基于客观数据（风格、酒精度、
  苦度）和声明的偏好。它不能替代
  正式场合中的人类侍酒师。

- **不处理其他饮品。** 烈酒、鸡尾酒、利口酒、
  汽水、无酒精饮料——任何非啤酒或葡萄酒的饮品
  均被忽略。

## 建议（可选）

**drinks-sommelier** 可与你的代理原生支持的
任何网络搜索工具配合使用。如果你的代理已具备可靠的网络搜索能力，
你无需额外配置。

但是，如果你希望每次都能进行彻底、防反爬虫的搜索
（尤其当产品页面受到反爬系统保护时），
考虑安装配套的
**[browser-search](https://github.com/Johell1NS/browser-search)** 技能。
它能同时搜索数十个引擎，帮助代理在标准工具
难以奏效时找到产品信息。
