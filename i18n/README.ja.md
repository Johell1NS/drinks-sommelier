## 存在理由

レストラン、パブ、酒屋、スーパーマーケット：問題はいつも
同じです。**数十の選択肢があり、どれが本当に自分の
好みに合うか確信が持てない。**

ウェイターに尋ねたり、スマホで5つのタブを開いたり、
一番綺麗な名前が最高のものだろうと願ったりすることもできます。あるいは、
AIエージェントに尋ねることもできます — そしてエージェントはあなたの好みを
すでに知っています。

**drinks-sommelier** は、あらゆるAIエージェントに個人の
ソムリエとしての役割を教えます。エージェントは：

1. あなたの味の好みを一度だけ学習します
2. あらゆる入力を分析します：棚の写真、デジタルメニュー、ユーザーが
   書いた名前のリスト
3. 各製品に関する最新情報を検索します
4. 各製品をあなたの味覚プロファイルと比較します
5. 何を選ぶべきか、そしてその理由を正確に伝えます

すべて透明で、正直で、無料で、何も捏造することなく。

## 利点

- **時間とともにあなたの味覚を学習。** スキルを使えば使うほど、エージェントは
  あなたの味覚を理解します。各インタラクションがプロファイルを洗練させます。

- **ビールとワインが1つのスキルに。** 2つの別々のスキルを
  インストールする必要はありません。エージェントは両方のカテゴリを処理し、
  分析対象を自動的に認識します。

- **画像とテキストからの分析。** 棚、メニュー、ワインリストの写真を
  撮影するか、リストを書き写します。エージェントは関連する
  製品（ビールとワイン）のみを抽出し、その他はすべて無視します。

- **幻覚なし。** エージェントは自身の事前トレーニング知識を
  使用することが**絶対に禁止**されています。各製品は
  評価前に常にウェブで即座に調査されます。

- **選好指数 0-100%。** 各提案には、その製品があなたの好みに
  どの程度合うかを示す明確なスコアが付いています。
  中途半端はありません。

- **永続的な味覚プロファイル。** あなたの好みは
  読み取り可能で編集可能なテキストファイルに保存されます。
  セッション間で失われることはありません。

- **エージェント主導のセットアップ。** 初回は、エージェントが
  適切な質問をしてあなたの好みを理解し、手動設定に
  直面させることはありません。

- **オープンソース、MITライセンス。** 無料、サブスクリプション不要、
  APIキー不要、制限なし。使用、変更、配布が可能です。

- **あらゆるAIエージェントで動作。** OpenCode向けに書かれていますが、
  OpenClaw、Hermes Agent、Claude Code、Cursor、
  GitHub Copilot、その他あらゆるエージェントに簡単に変換可能です。
  このREADMEをエージェントに提示するだけで、簡単かつ自動的に変換されます。

- **完全にカスタマイズ可能。** SKILL.mdはテキストファイルです。
  ルールを追加し、基準を変更し、ワークフローに適応させてください。

## 仕組み

プロセスは5つのフェーズに分かれており、クエリのたびに
エージェントが順番に実行します。

### フェーズ1 — 味覚プロファイルの設定（初回のみ）

スキルが初めて読み込まれるとき、エージェントは SKILL.md 内の
「ユーザーのビールの好み」と「ユーザーのワインの
好み」の段落がすでに記入されているかどうかを確認します。

記入されていない場合、エージェントは `data/SETUP.md` を開きます。このファイルには以下が含まれます：
- 好みを収集するためのガイド質問（甘い/苦い、アルコール度数、
  好きなスタイルと嫌いなスタイルなど）
- 正しく記入された段落の例
- データが入力された `data/` ファイルの例

エージェントは質問をし、回答を収集し、SKILL.md の2つの
味覚段落を修正します。セットアップ中にユーザーが
特定の製品に言及した場合、エージェントはそれを対応する
`data/` ファイルに記録します（オプションですが、
プロファイルを洗練させるのに役立ちます）。

段落は手動で記入することもできます。

`data/known-preferred-beers.md` と `data/known-preferred-wines.md`
はオプションであり、過去に評価した特定の製品が含まれます。
これらが完全であればあるほど、エージェントはあなたのニュアンスを理解します。

### フェーズ2 — 入力分析

エージェントは、あなたが何をどのような形式で提供したかを特定します：

- **テキスト**：書かれたリストからビールとワインの名前を抽出します
- **画像**：写真（棚、メニュー、ワインリスト）を分析し、
  ビールまたはワインである製品のみを認識します。
  完全に無視するもの：スピリッツ、リキュール、軽食、料理、
  デザート、清涼飲料水、アクセサリー。ビールやワイン以外はすべて無視します。
- **混合**：テキストと画像の両方を送信した場合、すべてを分析します。

入力が明確でない場合、エージェントは先に進む前に質問します。

### フェーズ3 — ウェブ調査（幻覚防止）

エージェントは製品を評価するために自身の事前トレーニング知識を
使用することが**絶対に禁止**されています。その知識は
古くなっていたり、不正確だったり、さらに悪いことに捏造されている可能性があります。
そのため、特定された製品のすべてについて対象を絞ったウェブ検索を
実行します。

エージェントが検索する内容：

| カテゴリ | 収集する情報 |
|---|---|
| **ビール** | スタイル、アルコール度数、甘さ、苦さ(IBU)、ボディ、アロマノート、原材料 |
| **ワイン** | ブドウ品種、原産地呼称、生産地域、ヴィンテージ、ボディ、酸味、タンニン、アルコール度数、甘さの特徴 |

製品がオンラインで見つからない場合、エージェントは正直にその旨を
伝え、特徴を捏造しません。

### フェーズ4 — 評価

エージェントは各製品をあなたの味覚プロファイルと比較します。プロファイルには
2つの異なるレベルがあります：

1. **味覚ルール** — 味覚の段落に書かれた一般的なガイドライン
   （例：「甘いビール」、「甘すぎない
   ワイン」、「アルコール度数 ≤ 9°」）。これらは拘束力があります。
2. **data/ ファイル内の製品** — あなたが好きだった、または
   嫌いだったビールとワインの具体的な例。これらは味覚ルールを
   調整するのに役立ちます（例：ビールXが好きでYが嫌いな場合、
   エージェントはあなたの言う「甘い」と「苦い」の意味を理解します）。

エージェントは**0 から 100% の選好指数**を割り当てます：

| 指数 | 意味 |
|---|---|
| 90-100% | あなたの好みに完璧な製品 |
| 70-89% | 優秀、わずかな不一致 |
| 50-69% | 許容範囲だが最適ではない |
| 30-49% | あなたの好みにあまり合わない |
| 0-29% | 避けるべき |

食事のコンテキストが指定された場合、エージェントは
フードペアリングも評価します（副次的な優先順位）。

### フェーズ5 — 推奨

エージェントは回答を明確に構造化します：

1. **第一候補**：選好指数付きの最適な製品、
   およびそれがあなたの好みに適している理由の説明
2. **代替案**（ある場合）：選好の高い順
3. **ペアリング**：製品のフードペアリング提案
4. **避けるべき製品**：あなたの好みに明らかに
   合わないものとその説明

## インストール

スキルは2つの方法でインストールできます：

### npx skills を使用（推奨）

```bash
npx skills add Johell1NS/drinks-sommelier --skill drinks-sommelier
```

### git clone を使用

```bash
git clone https://github.com/Johell1NS/drinks-sommelier.git
```

> **注意：** `npx skills add` とは異なり、git clone は
> エージェントのスキル保存場所を知りません。スキルを
> エージェントの正しいパスに配置する必要があります
> （例：OpenCode は `~/.config/opencode/skills/` を使用、
> 他のエージェントは異なる場合があります）。配置すると、
> エージェントが自動的に検出します。

## 設定

スキルが機能するには、あなたの好みを知る必要があります。
2つの方法で設定できます：

### ガイド付き設定（推奨）

スキルをインストールした後、エージェントに「
drinks-sommelier スキルを設定して」などと依頼してください。
エージェントは SKILL.md を読み込み、まだ記入されていない味覚段落を
見つけ、`data/SETUP.md` を使用して初期設定を案内します。

または、単にビールやワインの推薦を依頼してください：
エージェントはスキルが初期化されていないことを自動的に検出し、
適切な質問をします。

### 手動設定

`SKILL.md` を開き、次の段落を記入してください：

**ユーザーのビールの好み**
```
甘くて苦くないビールが好きです。
アルコール度数は 9% を超えてはいけません。
ベルギースタイルとフルーティーなビールが好きです。
IPA、スタウト、苦すぎるビールは嫌いです。
```

**ユーザーのワインの好み**
```
甘すぎない赤ワインが好きです。
白ワインは、ヴェルメンティーノとソーヴィニヨンが好きです。
酸味と爽やかさのあるワインが好きです。
パシートや酒精強化ワインは嫌いです。
```

また、`data/known-preferred-beers.md` と
`data/known-preferred-wines.md` に、すでに試して
評価した特定の製品を記入することもできます。この部分はオプションですが、
プロファイルを洗練させるのに役立ちます。

**例 — `data/known-preferred-beers.md`**
```
# 好きなビール
- Kwak
- Triple Karmeliet
- La Trappe Blond
- Flea Margherita

# 嫌いなビール
- IPA全般
- スタウト
- アルコール度数 9% 以上のビール
```

**例 — `data/known-preferred-wines.md`**
```
# 好きなワイン
- Vermentino Costamolino
- Syrah Soraia Casale Valle Chiesa
- Rebeca Firriato

# 嫌いなワイン
- パシートワイン
- 酒精強化ワイン
```

### 経時的な更新

製品について評価を下すたびに（「これが好き」「これは好きじゃない」）、
エージェントは `data/` ファイルを自動的に更新します。
一方、SKILL.md の段落にある味覚ルールは、将来の評価に
拘束力を持つため、あなたの明示的な確認があった場合にのみ
更新されます。

## 使用例

### シナリオ1 — ワイン棚（画像）
酒店の棚の写真を撮ります。エージェントがワインを特定し、
すべてをウェブで調査し、あなたのプロファイルと比較します。
どれを選ぶべきか、その理由、そして何と合わせるべきかを教えてくれます。

### シナリオ2 — ビールメニュー（画像）
パブのビールメニューの写真を撮ります。エージェントはカクテル、
スピリッツなどを無視し、ビールのみを分析し、各ビールの情報を
検索します。あなたの好みに最も合ったビールを提案します。

### シナリオ3 — 混合テキストリスト
「これらのワインがあります：Chianti Classico、Vermentino Costamolino、
Amarone。そしてこれらのビール：Kwak、地元のクラフトIPA、Leffe Blonde。」
と書きます。エージェントは両方のカテゴリを評価し、
全体的に最適な選択肢を教えてくれます。

### シナリオ4 — リストなしでのリクエスト
「ディナー用にワインを買わないといけないんだけど、何がおすすめ？」
エージェントは捏造しません。何が手に入るか、どんなディナーか、
そしてすでに考えているワインがあるかどうかを尋ねます。

### シナリオ5 — 新しい好みの表明
提案の後で「あのビール、本当に気に入った」と言います。
エージェントはそれをあなたのお気に入りに追加し、次回は
さらに正確になるようプロファイルを更新します。

## このスキルがやらないこと

- **プロのテイスティング評価は提供しません。** 提案は
  客観的データ（スタイル、アルコール度数、
  苦味）と宣言された好みに基づいています。正式な場面で
  人間のソムリエに代わるものではありません。

- **他の飲料は扱いません。** スピリッツ、カクテル、リキュール、
  清涼飲料水、ノンアルコール飲料 — ビールやワイン以外は
  無視されます。

## 提案（オプション）

**drinks-sommelier** は、エージェントがネイティブでサポートする
あらゆるウェブ検索ツールで動作します。エージェントがすでに信頼性の高い
ウェブ検索機能を持っている場合、追加のものは必要ありません。

ただし、毎回徹底したアンチボット耐性のある検索を
確保したい場合（特に製品ページがアンチボットシステムの背後にある場合）、
コンパニオンスキル
**[browser-search](https://github.com/Johell1NS/browser-search)** の
インストールを検討してください。これは数十のエンジンを同時に検索し、
標準ツールが困難な場合でもエージェントが製品情報を見つけるのを支援します。
